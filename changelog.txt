Version 1.10.0 — April 5, 2026

New:
Live character count displayed below the output area — shows the number of characters in the transformed output, styled as unselectable dim text
Undo, redo, and clear buttons below the input textarea — undo/redo track a custom history stack so clearing the input can be undone to restore text

Version 1.9.0 — April 5, 2026

New:
Bold number formatting: digits 0–9 can be styled independently with Bold, Sans-Serif Bold, Double-Struck, or Monospace Unicode number variants
Backtick no-format zones: wrapping text in backticks (e.g. `raw text`) exempts it from all transformations — markdown, number styling, punctuation, spaces, symbols, everything
Symbol picker: "Random" mode replaced with "Pick" mode — a scrollable grid showing all available symbols with per-symbol enable/disable toggles and bulk All/None buttons
Symbol picker state persists in profiles and syncs with the settings modal

Version 1.8.0 — March 26, 2026

New:
Discord-style formatting: _italic_ (single underscore), ***bold italic***, and ~~strikethrough~~ support
Nested formatting combinations: __*underline italic*__, __**underline bold**__, __***underline bold italic***__
Bold italic uses dedicated sans-serif bold italic Unicode font (𝙖𝙗𝙘)
Strikethrough uses Unicode combining long stroke overlay (U+0336)

Changes:
Profile storage migrated from cookies to localStorage — existing profiles are automatically migrated
Profile limit increased from 5 to 8
Profile dropdown separator shortened to fix mobile wrapping issue

Fixes:
Import validation now sanitizes profile names (strips control characters, zero-width chars) and validates field types
Imported profiles are stripped of unknown keys to prevent prototype pollution
Text transformation pipeline now wrapped in try/catch — errors fall back to raw input instead of blank output
Save notification not appearing on PC — duplicate function override was silently failing
Settings slider not draggable on PC — base CSS padding was collapsing the input's hit area
Quote style no longer replaces single quotes/apostrophes — prevents mangling words like don't and wouldn't
